## Neena Slack Bot — Session Summary (copy from here)

### Project
**Repo:** `slack_intermediary` (GitHub: `bp134/slack_intermediary`, branch `main`)  
**Purpose:** Belfield Pharmacy Slack bot (“Neena”) — channel-scoped task memory, PII pseudonymization on disk, Socket Mode (no HTTP).

---

### What’s built and working

**Core bot (Python + slack-bolt, Socket Mode)**
- Detects tasks from keywords (follow up, owing, order, task, update, urgent, etc.)
- Pseudonymizes PII before saving (Presidio + spaCy `en_core_web_sm`)
- Confirms tasks in-channel with real names; stores pseudonyms on disk
- `done` / `completed` marks latest open task in channel
- `show master list` → DM to full workspace members, de-pseudonymized
- Admin: `emergency stop` / `resume bot`
- `help` when LLM is off
- Per-channel JSON memory + `master_tasks.csv` + SQLite `history.db` on `/data`

**Privacy model**
- **Disk/logs/OpenRouter:** pseudonyms only  
- **Slack (in-channel + master list DM):** real names for workspace staff (GDPR/DSPT aligned)

**Deployment (Render)**
- **Background Worker** (not Web Service)
- **Python 3.12.8** (`python_version` / `PYTHON_VERSION`)
- **Persistent disk** at `/data`, **`DATA_PATH=/data`**
- **Build:** `pip install --upgrade pip && pip install -r requirements.txt`
- **Start:** `python app.py`
- **`render.yaml`** in repo (Blueprint optional; service was set up manually)

**Required env vars**
- `SLACK_BOT_TOKEN` (`xoxb-`)
- `SLACK_APP_TOKEN` (`xapp-`, `connections:write`)
- `MY_SLACK_USER_ID`
- `DATA_PATH=/data`
- `LLM_ENABLED=false` (default — task-only mode)
- `OPENROUTER_API_KEY` only if `LLM_ENABLED=true`

**Slack scopes:** `chat:write`, `channels:history`, `groups:history`, `im:history`, `im:write`, `users:read`

**Status:** Deployed and **working** — task capture, confirmations, master list tested successfully.
